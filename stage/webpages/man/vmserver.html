vmserver

Name

vmserver -- VOCAL voice mail control server

Synopsis

vmserver [options] [sip-port]

Description

vmserver controls the voicemail system by controlling a voice mail user agent (UA) which actually handles the SIP calls. vmserver knows about things like answering the call, which greeting to play, and where to send the .wav file which contains the caller's voice mail message.

Options

-d

fork into background as a daemon(NOTE: this is opposite all other commands)

-i

run from inetd

-f arg

specify voicemail configuration file

-h arg

specify number of VMCP handles

-v arg

specify the debugging level

-m

run in IVR mode. In this mode, vmserver listens on port 9024 instead of 8024. the corresponding option to ua is -a.

See Also

vocal(8)
